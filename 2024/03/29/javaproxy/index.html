# 🥳代理

什么是代理呢。举个🌰子，你要去买房子，但你并不知道房子的一些细节，比如地段，是否有升值空间，安保好不好等等。所以你可以找一个熟悉房地产产业的一个专家，也就是我们所说的中介来咨询如何买房子。他可以根据你的需求提供你具体的信息，然后你就可以判断买哪些房子。客户不会直接接触房子，而是直接访问中介，中介再提供房子的信息。这个中介就是我们所说的代理。在代理模式中，客户端不会直接接触目标对象，而是访问代理对象，代理对象再去访问目标对象。

在代理中有如下术语：

  • 目标 (类) 对象：被代理 (类) 对象代理的对 (类) 对象。可以理解为上述例子的房子
  • 代理 (类) 对象：代理目标 (类) 对象的 (类) 对象。可以理解为上述例子的中介

使用代理模有如下优点：

  • 提供安全访问。我们使用代理对象来帮我们过滤客户端的非法请求，从而保证了系统安全。
  • 代码增强。我们可以在代理对象中提供一些额外的功能。例如：日志，格式转换等等。
  • 低耦合。让业务类更加注重业务，一些非业务的代码则交给代理类实现。

为了保证代理类和目标类一一对应，也就是说，当我们访问代理类的时候，代理类能精确访问到目标类的某个功能，所以我们使用继承和实现约束代理类和目标类的关系来实现代理

# 🔥动态代理

jdk 提供了一种基于反射的动态代理方式来实现代理方式。该方法不用硬编码代理类，而是在内存中动态生成代理对象。实际上，动态代理仍然使用的是实现接口的方式来实现代理。实现 jdk 的动态代理需要如下的类：

  • Proxy ：代理类的基类，所以的代理类都继承于这个类
  • InvocationHandler ：调用处理器接口。用于执行目标类的方法。
